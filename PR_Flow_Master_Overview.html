Key Process Stages

  1. Creation & Validation: User creates PR with all required information. System validates before saving.
  2. Procurement Review (SUBMITTED): Only Procurement (Level 3) or Admin (Level 1) can take actions. Can edit most fields. Four outcomes: Move to Queue, Request Revision, Reject, or Cancel (requestor only).
  3. Queue Processing (IN_QUEUE): Validates quotes against thresholds, ensures proper approver is selected.
  4. Approval Process (PENDING_APPROVAL): Designated approver reviews and makes decision (Approve, Reject, or Request Revision).
  5. Revision & Resubmission (REVISION_REQUIRED): Only Requestor or Admin (Level 1) can take actions. Requestor edits PR and resubmits, returns to Procurement Review.
  6. Fulfillment (APPROVEDâ†’ORDEREDâ†’COMPLETED): Order is placed, items are received, PR is completed.
  7. REJECTED & CANCELED States: PR ends as REJECTED (procurement/approver) or CANCELED (requestor). Can be resurrected: REJECTED â†’ restored by Procurement/Admin to highest previous status; CANCELED â†’ restored by Requestor to SUBMITTED only.
